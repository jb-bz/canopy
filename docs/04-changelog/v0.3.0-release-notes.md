# canopy v0.3.0 — multi-provider, multi-client setup

**MIT-licensed Python treedocs clone.** `canopy` now ships with a real `setup` command that works with the most popular LLM providers AND the most popular MCP clients — no more "edit your config by hand" or "edit `cli.py` to change providers."

## What's new since v0.2.0

- **`canopy setup`** — interactive TUI for first-run config. Prompts for provider, model, base URL, MCP client, and API key.
- **`canopy setup --non-interactive`** — scriptable flags for CI / dotfile repos.
- **Two LLM providers ship in-box:**
  - **Anthropic Messages API** (default). Works for Anthropic, MiniMax (your current setup), GLM, and any Anthropic-compatible endpoint.
  - **OpenAI Chat Completions API**. Works for OpenAI, Mistral, Groq, Together, Ollama (local), LM Studio, vLLM, OpenRouter, LiteLLM proxy.
- **Seven MCP clients supported out of the box:**
  - Claude Code, OpenAI Codex, OpenCode, Cline, Continue, Cursor, Windsurf
- **Anything else gets the generic doc.** Run `canopy setup --non-interactive --client clawbot` (or `nanobot`, or any agent name) and it prints a self-contained markdown doc with a paste-to-your-agent prompt plus manual-config instructions.
- **`canopy fill --provider openai`** lets you switch providers per-call without re-running setup.
- **Refactored**: `canopy.llm.call_minimax` is now a back-compat wrapper around the new `Provider` abstraction. Old code keeps working.

## Usage

### First-time setup (interactive)

```sh
canopy setup
# Walk through provider / model / base URL / client / API key.
# Writes the right config file for the chosen client.
```

### First-time setup (scriptable)

```sh
canopy setup --non-interactive \
    --provider anthropic \
    --model MiniMax-M3 \
    --client claude-code \
    --api-key "$BWS_MINIMAX_SECRET_ID" \
    --global-config
```

### Switch to OpenAI without re-running setup

```sh
canopy --root /path/to/repo fill --provider openai --model gpt-4o-mini
```

### Help an unknown MCP client register canopy

```sh
canopy setup --non-interactive --provider anthropic --client clawbot
# Prints a markdown doc you can paste to clawbot, nanobot, or any agent.
```

## What's covered

| Provider | Endpoint | Default model |
|---|---|---|
| `anthropic` | `/v1/messages` (Anthropic Messages) | `MiniMax-M3` |
| `openai` | `/v1/chat/completions` (OpenAI Chat Completions) | `gpt-4o-mini` |

| MCP client | Config file | Format |
|---|---|---|
| `claude-code` | `~/.mcp.json` | JSON |
| `codex` | `~/.codex/config.toml` | TOML |
| `opencode` | `~/.config/opencode/config.json` | JSON |
| `cline` | `~/Library/Application Support/Code/.../cline_mcp_settings.json` | JSON |
| `continue` | `~/.continue/config.json` | JSON |
| `cursor` | `~/.cursor/mcp.json` | JSON |
| `windsurf` | `~/.codeium/windsurf/mcp_config.json` | JSON |
| any other | prints `docs/setup/UNKNOWN_CLIENT.md` | markdown |
