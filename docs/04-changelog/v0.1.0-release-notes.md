# canopy v0.1.0 — `fill` subcommand goes live

**MIT-licensed Python treedocs clone.** All 6 CLI subcommands are now functional; the only stub in v0.0.1 (`fill`) is now fully wired.

## What's new since v0.0.1

- **`canopy fill`** is fully implemented. Fetches the MiniMax API key via `bws`, calls the LLM in batches of 50 paths, parses JSON responses (with regex fallback on malformed output), and writes the results into `canopy.yaml`.
- New modules:
  - `canopy/bws.py` — `fetch_secret()` + typed `BwsError`
  - `canopy/llm.py` — `call_minimax()` + typed `LlmError`
  - `canopy/fill.py` — `fill_missing()` with batching + JSON + regex fallback
  - `canopy/hindsight.py` — best-effort `retain()` to a Hindsight bank
- New flags on the top-level `canopy` command: `--base-url`, `--model`, `--bws-secret`, `--hindsight-url`, `--hindsight-bank`.
- New flags on `canopy fill`: `--batch`, `--max-words`, `--dry-run`, `--retain-hindsight`.
- **Security:** `bws` secret UUID is now read from `BWS_MINIMAX_SECRET_ID` env var instead of hardcoded.

## Usage

```sh
export BWS_MINIMAX_SECRET_ID=<your-uuid>
source ~/.config/bws/env
canopy --root /path/to/repo fill                  # fill all missing descriptions
canopy --root /path/to/repo fill --dry-run        # preview without burning LLM tokens
canopy --root /path/to/repo fill --retain-hindsight  # log to Hindsight bank
```
