# canopy v0.2.0 — automatic context discovery for `fill`

**MIT-licensed Python treedocs clone.** `canopy fill` now auto-discovers context docs from your repo instead of relying on a hardcoded list at the repo root.

## What's new since v0.1.0

- **Context discovery is automatic.** No more hardcoded tuple of root files. `fill` now walks the tree and finds:
  - **Root files** by basename glob: `README*`, `CHANGELOG*`, `AGENTS.md`, `CLAUDE.md`, `CONTRIBUTING*`, `package.json`, `pyproject.toml`, `Cargo.toml`, `go.mod`, `setup.py`
  - **`docs/**/*.{md,rst}`** at any depth — so `docs/00-spec/PLAN.md`, `docs/research/sub/deep.md` all flow to the LLM
  - **Module dirs** (`src/`, `lib/`, `app/`, `internal/`, `pkg/`) — any `*.md` inside
- **20 KB total context cap** (up from 8 KB). Per-file cap stays at 3 KB.
- Honors `.gitignore` + `STANDARD_EXCLUDES` — `node_modules/docs/foo.md` is correctly skipped.
- New module `canopy/context.py` is independently testable and reusable.

## Discovery example

Against a fixture repo with `docs/`, `src/auth/`, `node_modules/`, the LLM now sees:

```
AGENTS.md, CHANGELOG.md, README.md, pyproject.toml,
docs/00-spec/PLAN.md, docs/01-research/findings.md,
docs/architecture/sub/deep.md,
src/auth/README.md, src/billing/DESIGN.md
```

9 files, all relevant. `node_modules/**` is silently skipped.
