# canopy v0.4.0 — 6 providers + BYO + per-provider docs

**MIT-licensed Python treedocs clone.** This release makes canopy usable for **anyone with an LLM API key**, not just MiniMax/Anthropic users. Six built-in providers ship in-box, plus a Bring-Your-Own hook for vendors we don't cover.

## What's new since v0.3.0

- **Six LLM providers** — all with the same `complete(system, user) -> str` interface:
  - **OpenRouter** (default) — one key, every model. Free-form model names.
  - **Anthropic** — direct Anthropic API or Anthropic-compat (MiniMax, GLM).
  - **OpenAI** — Chat Completions API. Works for OpenAI, Mistral, Groq, Together, Ollama, LM Studio, vLLM.
  - **Gemini** — Google Generative Language API.
  - **Bedrock** — AWS Bedrock with inline SigV4 signing. **No boto3 dependency.**
  - **Cohere** — v2 chat API.
- **BYO (Bring Your Own)** — `BYOProvider(complete_fn=...)` lets you wire any HTTP API in 30 lines.
- **Per-provider docs** in `docs/providers/`:
  - Endpoint, headers, request/response body shapes, setup instructions, model list link.
  - One Markdown file per provider + a `BYO.md` for custom adapters.
- **AWS SigV4 implemented in ~30 lines of stdlib** (`hmac` + `hashlib`). Full unit tests. Bedrock works without `boto3`.
- **OpenRouter as new default provider** (per user preference). One key gives access to Claude, GPT-4, Gemini, Llama, DeepSeek, Mistral, and 100+ others.
- **Free-form `--model` field** — canopy never validates against a vendor's catalog. New models work on day one of vendor release.
- **Repo banner image** at `docs/assets/canopy-banner.png`.

## Usage

### First-time setup

```sh
canopy setup
# Walk through provider / model / MCP client / API key.
```

### Direct flag use

```sh
# OpenRouter
canopy fill --provider openrouter --model anthropic/claude-3.5-sonnet

# OpenAI
canopy fill --provider openai --model gpt-4o-mini

# Gemini
canopy fill --provider gemini --model gemini-2.0-flash-exp

# Bedrock (needs AWS creds)
AWS_ACCESS_KEY_ID=AKIA... AWS_SECRET_ACCESS_KEY=*** \
    canopy fill --provider bedrock --model anthropic.claude-3-sonnet-20240229-v1:0
```

### BYO example

```python
from canopy.providers import BYOProvider

def my_complete(system: str, user: str) -> str:
    import urllib.request, json
    req = urllib.request.Request(
        "https://api.example.com/v1/chat",
        data=json.dumps({"system": system, "user": user}).encode(),
        headers={"authorization": "Bearer MY_KEY"},
        method="POST",
    )
    with urllib.request.urlopen(req, timeout=60) as resp:
        return json.loads(resp.read())["text"]

provider = BYOProvider(complete_fn=my_complete)
# Pass to canopy.fill.fill_missing(provider=..., ...)
```

## Future: standalone providers library

The `Provider` Protocol + all built-in adapters are intentionally clean — they could be extracted into a standalone `canopy-providers` PyPI package and reused by other tools (chat UIs, eval harnesses, code analyzers). The BYO provider demonstrates that the contract is minimal: 30 lines of Python.
